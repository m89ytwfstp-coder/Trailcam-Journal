## Issue 1

**Title:** `[Bug] macOS: deleted entries don't clean up image files`

**Labels:** `bug`, `macOS`

**Body:**
```
## Description
In `EntryStore.cleanupEntry()`, the image file deletion is wrapped in `#if os(iOS)`:

```swift
private func cleanupEntry(_ entry: TrailEntry) {
#if os(iOS)
    if let filename = entry.photoFilename, !filename.isEmpty {
        ImageStorage.deleteJPEGFromDocuments(filename: filename)
    }
#endif
}
```

On macOS, images imported via `MacImportEntries` are saved to the Documents directory by `MacImageStore.saveDownsampledJPEG()`, but they are **never deleted** when the entry is removed. Over time this causes orphaned JPEG files to accumulate silently.

## Fix
Add a macOS branch to `cleanupEntry()` that calls the equivalent `MacImageStore` file deletion when `entry.photoFilename` is set.
```

## Issue 2

**Title:** `[Bug] DateFormatter recreated on every import call in MacImportEntries`

**Labels:** `bug`, `performance`, `macOS`

**Body:**
```
## Description
In `MacImportEntries.swift`, `parseExifDate(_ raw: String)` creates a new `DateFormatter` instance on every invocation:

```swift
private func parseExifDate(_ raw: String) -> Date? {
    let formatter = DateFormatter()
    formatter.locale = Locale(identifier: "en_US_POSIX")
    formatter.timeZone = .current
    formatter.dateFormat = "yyyy:MM:dd HH:mm:ss"
    return formatter.date(from: raw)
}
```

`DateFormatter` is expensive to initialise. When importing a large batch of photos this is called once per image, causing unnecessary overhead.

## Fix
Convert `formatter` to a `private static let` so it is created once and reused.
```

## Issue 10

**Title:** `[Improvement] Migrate entry persistence from UserDefaults to file-based JSON`

**Labels:** `enhancement`, `data`

**Body:**
```
## Description
`EntryStore` currently persists all entries as a single JSON blob in `UserDefaults`:

```swift
UserDefaults.standard.set(data, forKey: storageKey)
```

`UserDefaults` has an effective size limit (~4 MB depending on platform) and is not designed for large structured data. As the user accumulates entries with metadata, this will become a problem.

## Goal
Move persistence to a dedicated JSON file in the app's Application Support directory (or use SwiftData). The migration should be automatic on first launch — read from UserDefaults if the file doesn't exist yet, write to the new location, then clear the UserDefaults key.
```
